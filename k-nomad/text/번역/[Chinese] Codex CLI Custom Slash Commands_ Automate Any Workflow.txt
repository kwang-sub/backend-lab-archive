大家好，这段视频要介绍 Codex CLI 里一个非常强大却经常被忽视的功能：自定义斜杠命令。屏幕上可以看到很多内建命令，我们等会儿会演示其中几条，不过真正的威力在于你能为任何任务或流程安装自己的命令，从而节省大量时间并搭建高效的工作流。看完本片之后，你就会明白这些命令的运行方式，并且能够从零创建第一条自定义斜杠命令。

如果你还不熟悉斜杠命令或者刚接触 Codex CLI，先来看一下系统预装的命令。列表里列出了所有可用命令以及它们负责的工作。界面顶部显示当前正在使用的模型，这里是 GPT-5 Codex high。要切换模型，可以调用 `/model` 命令：按下斜杠键调出命令面板，选择 model 后回车，就会出现模型选择界面，同时还可以选择推理层级（低、中、高），我们在示例中选的是 high。`/approvals` 也是常用命令，可以切换审批模式：Level 1 仅能读取，Level 2 自动批准，允许读取文件、修改文件并运行命令，Full Access 则能修改整台机器上的所有文件，而不仅限于当前项目。我们在视频中选择 Level 2。

自定义命令既可以安装在全局，也可以安装在项目层级。全局命令可在机器上的任意项目中调用；项目级命令只能在特定项目里使用。我个人偏好将常用命令装在全局目录里，因为很多指令可以跨项目复用。只有在公司或团队场景下，某些命令包含敏感数据、不适合在全局暴露时，才会改用项目层级。本视频的演示全部使用全局安装。

接下来聊聊如何安装自定义斜杠命令。方法不止一种，稍后我也会展示它的后台结构。我推荐的最快方法，是直接在 Codex 里开一个新的对话，把想做的事描述清楚，并说明要做成自定义斜杠命令，Codex 就会帮你创建所需文件并配置好。虽然也能手动构建，但让 Codex 代劳省时省力。装好之后，如果想确认有哪些命令，可以按斜杠输入 `prompts`，Codex 就会列出当前这台机器上的自定义命令。我目前只安装了 `/code-review`。

现在让 Codex 展示该命令背后的文件：`~/.codex/prompts/code-review.md`。这个目录存放全部全局自定义命令，每个命令都是一个 Markdown 文件。当我们执行某条命令时，Codex 会读取对应的 Markdown 文件，把里面的说明和上下文作为执行依据。以 `/code-review` 为例，它会读取 `code-review.md`，里面写着“对指定文件或目录做彻底的代码审查”，还列出关注点（代码质量与可读性、潜在漏洞或边界情况等）、评审流程以及输出格式。输出必须区分 Critical / High / Medium / Low 级别的问题，并注明代码中表现良好的部分。文件里还使用 `$1` 来代表命令后的第一个参数，用来指定要检查的文件或目录；如果没有提供参数，就按“审查当前目录所有代码文件”执行。这一点在大型代码库中尤为重要：如果只想检查特定文件，务必在命令后加上路径。

总的来说，自定义斜杠命令是为了把重复的任务或冗长的提示词自动化。把复杂提示写进 Markdown 文件后，就能用一条命令多次复用，再也不用每次都手动输入或复制粘贴。我们来实际跑一次 `/code-review`。我准备了一个包含多处问题的示例文件 `messy-code.js`，执行 `/code-review messy-code.js` 后，Codex 会读取刚才提到的 Markdown 作为上下文来分析该文件。完成后会按照 Critical、High、Medium、Low 的顺序列出问题，同时说明代码中的亮点。

紧接着我们可以回复 “请修复你发现的所有问题”，Codex 就会生成行动计划并修改 `messy-code.js`，完成后说明修复了哪些内容，还会给出后续建议。对于由 AI 生成或协助编写的代码来说，这种工作流非常实用，因为可以快速检查并修复潜在问题，保证代码质量。

我也会把常用命令做成命令包。例如我的网站 snapperai.io/codex-prompt-packs 提供了一些预制的 Codex CLI 包，你可以立即下载使用。我们重点演示 “Code Analyzer Pack”，它附带一段复制即用的提示词，一次性安装五条命令：`/code-review`、`/security-audit`、`/refactor-finder`、`/dependency-check`、`/generate-tests`。Security Audit 会扫描 SQL 注入等漏洞；Refactor Finder 会找出潜在的重构机会；Dependency Check 会提示过期依赖；Generate Tests 则自动为指定文件或目录生成测试。这个包的优势在于省去了自己摸索提示词的过程——这些提示词已经在真实项目中验证有效。只要输入姓名和邮箱，就能收到下载链接。

打开安装文件后可以看到说明：该包包含 5 条面向代码分析的强力命令，适用于任何语言，30 秒即可设置完成，而且已经在生产场景下打磨过。步骤 1 要求“复制下面的主安装提示词并粘贴到 Codex CLI 中”。从 `markdown` 这一行开始复制即可。提示词开头写着“请在我的 Codex prompts 目录中创建下面五条自定义斜杠命令”，这里的路径就是全局命令存放位置。每个命令都会以 Markdown 文件的形式保存，例如 `code-review.md` 对应 `/code-review`。在每个段落里，除了文件名，还详细描述了命令的目标、需要遵循的输出格式等。例如 `security-audit.md` 指示 Codex 执行全面安全检查、扫描 SQL 注入、环境变量泄漏、弱加密等问题，并要求对每个问题报告严重级别、位置、漏洞类型、风险说明以及修复建议。命令统一使用 `$1` 作为第一个参数来接收目标文件或目录，如有需要还会定义默认回退行为。提示词的最后一句是“确认五条命令均已成功创建”，复制范围到这里结束。

步骤 2 提醒在安装后重启 Codex。因为新命令只有在重启 CLI 后才会进入命令列表。把刚才复制的提示词粘贴进 Codex 会话并执行，大约 55 秒后 Codex 会在 prompts 文件夹里生成所有 Markdown 文件并逐一验证，最后提示“如需在其他机器使用，请同步这些文件”。重新开启会话并输入 `/prompts`，就能看到 `code-review`、`dependency-check`、`generate-tests`、`refactor-finder`、`security-audit` 等命令都已经可用了。

接下来测试 `/refactor-finder`。这条命令会扫描当前目录的全部代码，寻找可重构的机会并提供摘要。在示例项目中运行后，它工作了 58 秒，输出摘要包括：共识别 8 个机会，估算技术债约 6.5 个开发者日，可立即处理的 Quick Win 有 3 项，并列出了“拉平校验流程”“合并折扣档位”“统一 HTTP 客户端使用方式”等建议。Critical Issues 区域突出显示 SQL 注入、数据库访问不安全、日志泄露秘密、弱口令哈希、支付日志记录卡号等问题，还给出了优先级更高的问题以及快速收益。回复结尾写着“如果需要样例重构草图或帮助排定顺序，请告诉我”。对于真实项目，特别是由 AI 编写的代码，这种命令可以帮助我们主动发现隐患。

最后演示 `/security-audit`。每当新增功能或文件，都应该运行这条命令，确认没有密码、API Key 等敏感信息泄露，也确保认证授权逻辑安全。我用 `auth-broken.js` 进行示范，命令只花了 14 秒就列出多项问题，包括严重级别、行号、漏洞类型（如 SQL Injection）、风险说明以及建议的修复方案和代码。回答末尾会提示“如果需要我帮你安全地重写模块或加入自动化密钥扫描测试，请告诉我”，于是我们继续让它“把这个模块安全地改写”，Codex 随即制定计划并修改文件。

完成修复后，再次对同一文件运行 `/security-audit`，这次的结果是“复检后未发现可被利用的安全漏洞”，还总结了当前安全状况以及已经到位的防护措施，并询问是否需要进一步的纵深防御。整个流程都在一场 Codex CLI 会话里完成：发现问题、修复问题、重新验证。我们无需重复输入冗长的提示词，只要调用已保存的命令，就能在大量文件上快速执行安全审计。
