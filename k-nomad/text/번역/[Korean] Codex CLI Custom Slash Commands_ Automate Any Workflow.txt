안녕하세요. 이번 영상에서는 Codex CLI에서 가장 강력하지만 아직 많이 쓰이지 않는 기능 중 하나인 맞춤형 슬래시 명령(custom slash command)을 소개하려고 합니다. 화면에서 보듯이 Codex에는 기본으로 제공되는 여러 슬래시 명령이 있어서 다양한 작업에 활용할 수 있습니다. 잠시 뒤에 몇 가지를 직접 살펴보겠지만, 진짜 힘은 원하는 작업이나 워크플로에 맞춰 맞춤형 슬래시 명령을 설치할 수 있다는 데 있습니다. 이렇게 하면 시간을 크게 아끼면서 강력한 워크플로를 구축할 수 있고, 영상이 끝날 때쯤이면 이 명령들이 어떻게 작동하는지 정확히 이해하고 처음부터 직접 만들어 볼 수 있게 됩니다.

먼저 슬래시 명령이 무엇인지 잘 모르거나 Codex CLI가 처음이라면, 기본으로 내장된 슬래시 명령을 간단히 짚고 넘어가겠습니다. 목록을 펼치면 사용 가능한 명령과 각 명령이 하는 일이 설명되어 있습니다. 화면 상단에는 현재 사용 중인 모델이 표시되는데, 여기서는 GPT-5 Codex high 모델을 쓰고 있습니다. 모델을 바꾸고 싶다면 `/model` 명령을 실행하면 됩니다. 슬래시(`/`) 키를 누르면 이용 가능한 모든 명령이 팝업으로 나타나고, 그중에서 model을 선택해 실행하면 모델 선택 창이 열립니다. 이어서 저희가 원하는 추론 레벨(저·중·고)도 고를 수 있는데, 지금은 high를 선택했습니다. 비슷한 방식으로 많이 쓰는 `/approvals` 명령을 실행하면 승인 단계를 변경할 수도 있습니다. 레벨 1은 읽기 전용, 레벨 2는 자동 승인으로 파일 읽기·수정·명령 실행이 허용되고, Full Access는 작업 중인 프로젝트를 넘어 로컬 머신의 모든 파일을 변경할 수 있게 해 줍니다. 이번에는 레벨 2를 선택했습니다.

명령을 설치할 수 있는 범위는 전역(Global)과 프로젝트(Project) 단위 두 가지가 있습니다. 전역으로 설치하면 내 머신의 모든 프로젝트에서 명령을 호출할 수 있고, 프로젝트 수준으로 설치하면 해당 프로젝트 안에서만 쓸 수 있습니다. 저는 반복해서 사용할 가능성이 높은 명령이 많기 때문에 기본적으로 전역 설치를 선호합니다. 다만, 기업 프로젝트처럼 특정 명령의 데이터가 민감해서 전역으로 노출하면 곤란한 경우에는 프로젝트 레벨 커스텀 명령을 쓰는 것이 좋습니다. 이번 영상에서는 모두 전역 명령으로 설치하겠습니다.

이제 맞춤형 슬래시 명령을 설치하는 절차를 살펴보겠습니다. 방법이 몇 가지 있는데, 일단 설치가 끝난 뒤 어떤 구조로 동작하는지 보여드릴 예정입니다. 제가 추천하는 가장 빠르고 쉬운 방법은 Codex와 직접 대화를 열어 “어떤 기능의 맞춤형 슬래시 명령을 만들고 싶다”고 설명하는 것입니다. 그러면 Codex가 필요한 파일을 생성하고 설정까지 알아서 처리해 줍니다. 수동으로 만드는 방법도 있지만, Codex에게 원하는 명령과 설치 위치를 알려주고 작업을 맡기는 편이 훨씬 빠릅니다. 설치가 제대로 되었는지 확인하고 싶다면 슬래시 창에서 `prompts`를 입력해 머신에 등록된 맞춤형 명령 목록을 열어 보면 됩니다. 저는 현재 `code-review`라는 커스텀 명령 하나만 설치해 둔 상태입니다.

이제 백엔드가 어떻게 구성되어 있는지 살펴보겠습니다. Codex에게 `codeexprompts/code-review.md의 전체 내용을 보여 달라`고 요청하면 됩니다. Codex는 `~/.codex/prompts` 디렉터리를 전역 커스텀 명령 저장소로 사용하며, 각 커스텀 명령은 하나의 마크다운 파일로 이루어져 있습니다. 특정 슬래시 명령을 실행하면 Codex가 해당 마크다운 파일을 불러와 명령을 수행하는 지침과 컨텍스트로 사용합니다. 예를 들어 `/code-review`를 호출하면 `code-review.md` 파일을 읽어 해당 지침에 따라 코드를 검토합니다. 실제 파일 내용을 확인해 보면 “지정된 파일 또는 디렉터리에 대해 철저한 코드 리뷰를 수행하라”는 설명과 함께 코드 품질·가독성, 잠재 버그나 엣지 케이스, 리뷰 절차, 출력 형식 등이 정의되어 있습니다. 출력 형식에는 Critical / High / Medium / Low 우선순위 문제를 나눠 보고하고, 잘 되고 있는 부분도 적도록 요구합니다. 또한 명령 실행 시 첫 번째 인자로 넘기는 파일·디렉터리를 `$1`이라는 자리 표시자로 지정했고, 만약 대상이 주어지지 않으면 “현재 디렉터리의 모든 코드 파일을 리뷰하라”는 폴백 규칙도 적혀 있습니다.

요약하자면 커스텀 슬래시 명령은 반복적인 작업이나 길고 복잡한 프롬프트를 자동화하기 위한 수단입니다. 자주 사용하는 프롬프트를 매번 타이핑하거나 복사해 올 필요 없이, 해당 프롬프트와 지침을 담은 마크다운 파일을 하나 만들어 슬래시 명령으로 등록해 두면 됩니다. 이제 `/code-review` 명령이 실제로 어떻게 동작하는지 테스트해 보겠습니다. 데모용으로 몇 가지 문제가 들어 있는 `messy-code.js` 파일을 준비해 두었고, `/code-review messy-code.js`를 실행하면 Codex가 방금 살펴본 마크다운 파일을 컨텍스트로 삼아 해당 파일을 점검합니다. 잠시 후 결과를 보면 Critical / High / Medium / Low 항목으로 이슈가 정리되어 있고, 코드에서 잘된 점도 함께 보고됩니다.

이제 Codex에게 “발견한 모든 문제를 고쳐 달라”고 후속 지시를 보내면, Codex가 작업 계획을 세우고 `messy-code.js` 파일을 수정합니다. 수정이 끝나면 어떤 변경을 했는지 알려주고, 필요하다면 추가 권장 사항도 제공합니다. 이렇게 코드를 검사하고 바로 고칠 수 있기 때문에, 특히 AI가 작성한 코드(vibe-coded 프로젝트)의 건강 상태를 정기적으로 점검하고 싶을 때 유용합니다.

제가 자주 사용하는 명령들은 팩 단위로 묶어서 설치하기도 합니다. 예를 들어 제 웹사이트 snapperai.io/codex-prompt-packs 에는 Codex CLI용으로 미리 만들어 둔 명령 팩이 있습니다. 여기에서 “Code Analyzer Pack”을 다운로드하면 다섯 가지 명령을 한꺼번에 설치할 수 있는 ‘복사 후 붙여넣기’ 프롬프트가 포함되어 있습니다. 이미 살펴본 `/code-review` 외에도 `/security-audit`, `/refactor-finder`, `/dependency-check`, `/generate-tests`가 들어 있습니다. Security Audit은 SQL 인젝션 같은 취약점을 점검하고, Refactor Finder는 리팩터링 기회를 찾아 주며, Dependency Check는 오래된 패키지를 알려 줍니다. Generate Tests는 지정한 파일이나 디렉터리에 대한 테스트 코드를 만들어 줍니다. 이 팩의 장점은 검증된 프롬프트를 한 번에 설치할 수 있어서, 매번 어떤 지침을 넣어야 할지 고민하지 않아도 된다는 점입니다. 이름과 이메일만 입력하면 설치용 마크다운 파일을 이메일로 받아 볼 수 있습니다.

설치 파일을 열어 보면 “이 팩에는 코드 분석을 위한 다섯 개의 강력한 명령이 포함되어 있으며, 어떤 언어 프로젝트에도 사용할 수 있고 30초면 설정이 끝난다”는 설명이 있습니다. Step 1에서는 “아래 마스터 설치 프롬프트를 Codex CLI에 붙여넣으라”고 되어 있고, `markdown`이라는 키워드부터 복사하면 됩니다. 프롬프트는 “내 Codex prompts 디렉터리에 아래 다섯 개의 커스텀 슬래시 명령을 만들어 달라”고 시작합니다. 여기서 사용되는 경로가 전역 명령 폴더이며, 각 명령은 해당 이름의 마크다운 파일 형태로 저장됩니다. 예를 들어 `code-review.md` 파일이 `/code-review` 명령의 지침을 담고 있습니다. 그 아래에는 각 명령이 어떤 목표를 가지고 어떤 출력 형식을 따라야 하는지 세부 지시가 포함되어 있습니다. `security-audit.md` 항목에서는 SQL Injection, 환경 변수 노출, 취약한 암호화 등을 검사하고, 발견된 문제마다 심각도·위치·취약점 유형·위험 설명·권장 수정사항을 보고하라고 명시되어 있습니다. 모든 명령은 `$1` 인자를 통해 대상 파일이나 디렉터리를 지정하도록 되어 있고, 필요하면 기본 폴백 동작도 정의되어 있습니다. 프롬프트 끝에는 “다섯 개 명령이 모두 성공적으로 생성되었는지 확인하라”는 문장이 있으며, 거기까지가 복사 범위입니다.

Step 2는 Codex를 재시작하라는 내용입니다. 새 명령을 설치한 뒤에는 항상 CLI 세션을 다시 시작해야 목록에 나타납니다. 실제로 설치 과정을 보면, 복사한 프롬프트를 Codex 채팅에 붙여넣고 엔터를 누르면 Codex가 약 55초 동안 다섯 개 명령을 모두 생성한 뒤 prompts 폴더에 저장하고 검증까지 마칩니다. 완료 메시지에서는 “필요하다면 다른 머신과 동기화하라”고 안내합니다. 이후 세션을 다시 열고 `/prompts`를 입력하면, 방금 만든 `code-review`, `dependency-check`, `generate-tests`, `refactor-finder`, `security-audit` 명령이 모두 목록에 표시됩니다.

이제 `/refactor-finder` 명령을 실행해 보겠습니다. 이 명령은 현재 디렉터리의 전체 코드베이스를 살펴보고 리팩터링 기회를 찾아 요약을 제공합니다. 데모 프로젝트에서 실행해 보니 약 58초 동안 분석을 수행했고, “총 8개의 기회를 발견했고, 예상 기술 부채는 약 6.5 개발자일, 바로 착수할 수 있는 Quick Win 3건”이라는 요약과 함께 세부 내역을 보여 주었습니다. Critical Issue로는 SQL 인젝션과 안전하지 않은 데이터베이스 접근, 비밀 값이 로그에 노출되는 문제, 약한 패스워드 해싱, 결제 플로 로그가 카드 번호를 남기는 문제 등이 나열되었습니다. 또한 높은 우선순위 항목과 빠르게 처리할 수 있는 개선 사항을 구분해 주고, 필요하면 리팩터링 스케치나 작업 순서를 제안해 주겠다고도 말해 줍니다. 실제 프로젝트, 특히 AI가 작성한 코드가 섞여 있어 품질이 불안할 때 이런 도구를 사용하면 잠재적 문제를 빠르게 찾아낼 수 있습니다.

다음으로 `/security-audit` 명령을 살펴보겠습니다. 이 명령은 새 기능을 만들거나 파일을 추가할 때마다 정기적으로 돌려야 할 정도로 중요합니다. 비밀번호나 API 키가 새어 나오지 않았는지, 인증·인가 로직이 안전한지 확인하는 데 유용합니다. 예제로 `auth-broken.js` 파일에 대해 명령을 실행했더니 14초 만에 여러 가지 문제를 보고했습니다. 출력에는 심각도, 문제 위치(라인 번호), 취약점 유형(예: SQL Injection), 위험 설명, 권장 수정 사항, 제안 코드가 포함되어 있습니다. Codex는 마지막에 “모듈을 안전하게 다시 작성하거나 비밀 탐지를 자동화하고 싶다면 도와주겠다”고 덧붙입니다. 그래서 바로 “이 모듈을 안전하게 다시 작성해 달라”고 지시하면, Codex가 계획을 세우고 파일을 수정합니다.

수정이 끝난 뒤 동일한 파일에 대해 다시 `/security-audit`을 실행하면, 이번에는 “검토 후 exploitable한 취약점이 발견되지 않았다”는 보고가 나오고, 현재 보안 상태와 이미 적용된 예방책을 짚어 주며 추가적인 방어 심화 조치가 필요하면 도와주겠다고 안내합니다. 이렇게 한 세션 안에서 문제를 찾고 고치고 재검증까지 할 수 있다는 것이 커스텀 슬래시 명령 워크플로의 진정한 힘입니다. 별도의 긴 프롬프트를 매번 작성할 필요 없이, 미리 만들어 둔 명령을 호출하기만 하면 되기 때문에 광범위한 파일에 대해 빠르게 보안 감사를 반복 실행할 수 있습니다.

마지막으로 마무리를 짓겠습니다. 커스텀 슬래시 명령에 대한 이번 소개 영상이 도움이 되었다면, 앞에서 언급한 code analyzer pack을 다운로드해 보세요. snapperai.io/codex-prompt-packs 페이지에서 이름과 이메일을 입력하고 버튼을 누르면, 다섯 개 명령을 한 번에 설치할 수 있는 마크다운 파일을 이메일로 받아 Codex CLI에서 즉시 적용할 수 있습니다. 물론 Codex CLI와 직접 대화하면서 자신만의 프롬프트를 정의해 설치할 수도 있습니다. 곧 PRD Wizard Pack을 활용해 PRD 생성을 자동화하는 방법을 다루는 영상도 올릴 예정이니 관심 있다면 구독해 주세요. 시청해 주셔서 감사합니다.
