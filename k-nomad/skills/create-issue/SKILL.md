# Create GitHub Issue

사용자가 전달한 주제, 문제 설명, 메모를 바탕으로 GitHub에 바로 등록할 수 있는 이슈 초안을 작성합니다.

## 목표

- 입력된 내용을 GitHub 이슈 문서 형태로 정리합니다.
- 문제 제기, 배경, 기대 결과, 작업 범위를 명확하게 드러냅니다.

## 작성 원칙

- 내용이 모호하면 일반적인 소프트웨어 프로젝트 기준에서 현실적인 가정을 세우고, 그 가정을 명시합니다.
- 입력이 버그에 가까우면 재현 절차, 실제 결과, 기대 결과를 우선 정리합니다.
- 입력이 기능 요청에 가까우면 배경, 목적, 사용자 가치, 작업 범위를 우선 정리합니다.
- 입력이 리팩터링이나 개선 요청에 가까우면 현재 문제, 개선 이유, 영향 범위를 우선 정리합니다.
- 불필요하게 장황하게 쓰지 말고, 팀이 바로 이해하고 착수할 수 있는 수준으로 작성합니다.

## 반드시 포함할 요소

- 제목
- 배경 또는 문제 설명
- 기대 결과
- 작업 범위
- 완료 조건
- 확인 또는 테스트 방법

## 추가 처리 규칙

- 필요한 경우 `비범위` 항목을 추가해 이번 이슈에서 다루지 않을 내용을 명시합니다.
- 필요한 경우 리스크 또는 확인이 필요한 사항을 `참고 사항`에 추가합니다.
- 구현 방법을 과도하게 고정하지 말고, 문제와 목표 중심으로 작성합니다.

## 출력 형식

아래 형식을 그대로 따릅니다.

### 제목
...

### 배경
...

### 기대 결과
...

### 작업 범위
- ...

### 완료 조건
- ...

### 확인 방법
- ...

### 비범위
- ...

### 참고 사항
- ...

입력이 매우 짧다면, 추정한 가정을 `참고 사항`에 분리해서 적습니다.
